初中数学学习文档生成计划

目标

把初中数学知识按“专题目录 → 具体学习文档”的方式长期沉淀。每个专题都要像“二次函数与抛物线”一样，不只是罗列知识点，而是包含图像、动画、题型方法、真实题目、解题步骤和复习入口。

最终形成一个可持续扩展的初中数学知识库：

  • 学科首页进入初中数学目录。
  • 初中数学目录按专题分类。
  • 每个专题是一篇独立 HTML 学习文档。
  • 每篇文档都能直接用于预习、复习和错题回看。

页面结构

当前结构：

  • topics/k12/math/index.html：在“已接入专题”里增加卡片。
  • topics/k12-roadmap.md：在“已接入专题”表格里增加一行。

单篇学习文档标准模板

每个专题页都建议包含这些模块：

模块目的要求
顶部导航明确当前位置返回数学目录，标明学段/专题
学习目标说明学完会什么3-5 条，面向学生语言
可交互动画先建立直观理解函数、几何、统计类都尽量有动态图或可调参数
知识框架建立地图用表格或卡片整理核心概念
推导过程解释为什么关键公式不能只给结论，要展示推导
题型速查快速选方法“看到什么条件 → 用什么方法”
例题精讲学会标准步骤每类题至少 1 道完整例题
常见坑点降低错题率汇总易错点和检查方法
综合题训练贴近考试至少 5 道真实风格题，带思路和步骤
总结口诀方便复习简洁，但不替代推理

动画与互动规范

不同专题采用不同互动方式：

专题类型推荐互动
函数Canvas 坐标系、参数滑杆、交点/顶点动态变化
几何SVG 动态标注、拖动点、角度/长度实时变化
方程与代数配方步骤展开、因式分解动画、数轴区间高亮
概率统计抽样模拟、频率变化动画、图表切换
综合题条件逐步显现、辅助线开关、解题流程折叠卡

每个互动模块必须有：

  • 默认状态，不操作也能看懂。
  • 一句观察提示，比如“先看开口，再看顶点”。
  • 移动端不溢出、不遮挡。
  • 不依赖复杂构建工具，保持静态 HTML 可发布。

生成顺序

阶段 1：函数主线

先完成函数相关专题，形成“坐标系 + 图像 + 参数 + 题型”的统一模板。

  1. 一次函数
  2. 一元二次方程
  3. 二次函数与抛物线复盘升级
  4. 函数几何综合

阶段 2：几何主线

几何文档重点是图形动态标注和辅助线思路。

  1. 平行线与角
  2. 三角形基础
  3. 全等三角形
  4. 相似三角形
  5. 圆

阶段 3：代数基础补齐

这些内容更适合用步骤展开、错因对比和题型卡来做。

  1. 有理数与数轴
  2. 整式与因式分解
  3. 分式
  4. 方程组与不等式

阶段 4：统计概率与压轴

最后补综合题和考试策略。

  1. 数据统计
  2. 概率初步
  3. 中考压轴题方法

每篇文档的题目配置

每篇专题至少包含：

  • 3 道基础例题：确认概念会用。
  • 3 道方法题：训练核心题型。
  • 3 道综合题：接近考试风格。
  • 1 道实际应用题：利润、面积、行程、桥洞、概率实验等。
  • 1 组易错辨析：选择题或判断题形式。

每道题都按这个格式写：

题目
破题点
思路
步骤
答案
易错提醒

质量验收标准

生成后逐项检查：

  • 页面能在本地静态服务器打开。
  • 从数学目录能点进去。
  • 页面内所有相对链接有效。
  • 动画或 SVG 在桌面和手机宽度不溢出。
  • 公式能正常渲染或至少可读。
  • 题目步骤没有跳步。
  • 每个专题都说明“什么时候用这个方法”。
  • 至少包含一个真实生活或考试综合场景。
  • GitHub Pages 部署后线上地址返回 200。

长期目标

形成一个完整的初中数学学习系统：

  • 学生可以按专题预习。
  • 家长可以按目录查找知识点。
  • 错题可以反向链接到专题。
  • 每个知识点都有动画、例题和综合题。
  • 后续可以继续扩展到小学数学和高中数学。
