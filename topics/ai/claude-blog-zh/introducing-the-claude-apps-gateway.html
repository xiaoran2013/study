面向 Amazon Bedrock 与 Google Cloud 的 Claude Apps Gateway 正式发布

原文标题：Introducing the Claude apps gateway for Amazon Bedrock and Google Cloud ｜ 发布日期：2026 年 6 月 29 日 ｜ 分类：Product announcements ｜ 产品：Claude Code ｜ 阅读时长：约 5 分钟 ｜ 查看原文

今天，我们正式推出面向 Amazon Bedrock 和 Google Cloud 的 Claude apps gateway。此前，在这些平台上运行 Claude Code，意味着要为每位开发者单独配置一份云凭证，手动把设置推送到每一台笔记本电脑，还要另外搭建一套工具来查看每位开发者的支出情况。这个网关是一个自托管的控制平面，为 Claude Code 提供企业级 SSO 登录、集中执行的策略、基于角色的访问控制，以及按用户划分的成本归因。

部署网关

该网关以单个无状态容器的形式部署在 Linux 上，由一个 PostgreSQL 数据库提供支撑。它保存你的上游凭证，依据你的身份提供商对开发者进行身份验证，分发并执行受管设置，并向你自己运营的采集器上报按用户划分的使用情况。为一名开发者开通权限，只需把他加入你的身份提供商（IdP）；解除权限，只需把他移除。

这个网关由 Anthropic 构建并发布，内置在开发者已经在安装的同一个 claude 二进制文件中，因此你可以在自己的基础设施上用一个无状态容器来运行它。因为网关和客户端是一起构建的，/login 流程能够感知网关的存在，客户端会在登录时自动应用受管设置，策略也能在每一次请求中得到一致的执行。

网关的运作方式

网关负责处理以下事项：

  • 身份验证。 它作为 OpenID Connect（OIDC）的依赖方，对接 Google Workspace、Microsoft Entra ID、Okta，或任何符合标准的 OIDC 提供商，并签发短期有效的会话。开发者的设备上不会留存长期有效的密钥。
  • 策略管理。 你可以在服务器端一次性定义受管设置，客户端会在登录时接收这些策略，网关会在每一次请求中执行它们。你可以集中调整允许使用的模型和默认设置。
  • 遥测数据。 客户端会为每一次请求打上使用量标记，网关通过 OTLP 把这些数据转发给你配置的采集器——运行在你自己的网络中，按你自己设定的保留周期存储。
  • 路由。 网关保存你的上游凭证，把推理请求路由到 Claude API、Amazon Bedrock 或 Google Cloud，并可以在不同服务商之间配置可选的故障转移。
  • 支出上限。 网关允许你设置每日、每周和每月的支出限额，限额可以按组织、按组或按用户来设定。

除非你把网关配置为使用 Claude API，否则它不会向 Anthropic 发送推理流量或使用数据。我们也在公开网关所使用的协议，这样其他网关开发者也可以实现同样的功能。

开始使用

该网关现已发布。要开始使用：

  • 部署网关： 下载 Claude Code CLI 二进制文件，把 gateway.yaml 指向你的 OIDC 发行方和上游凭证，并在你的 IdP 中注册一个 OIDC 应用。
  • 进行推广： 在客户端机器的 managed-settings.json 中配置 forceLoginMethod 和 forceLoginGatewayUrl 参数。客户端会在首次启动时连接到你的网关。
