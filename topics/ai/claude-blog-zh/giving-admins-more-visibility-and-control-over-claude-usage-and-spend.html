为管理员提供更强的 Claude 使用可见性与支出控制能力

原文标题：Giving admins more visibility and control over Claude spend ｜ 发布日期：2026 年 7 月 2 日 ｜ 分类：Product announcements ｜ 产品：Claude Enterprise ｜ 阅读时长：约 5 分钟 ｜ 查看原文

Claude Enterprise 现已上线全新的分析功能与成本控制工具。

我们正在为 Claude Enterprise 推出更丰富的管理员分析功能、模型级别的权限管理，以及支出提醒。随着 Claude 在组织内承接越来越复杂、越来越有难度的智能体工作，使用量和成本模式已经和标准聊天工具截然不同。这些新的控制能力，能让管理员既看清 Claude 的使用情况，又拥有管理成本的工具。

今天新增的功能，是在 Anthropic 已经提供的控制能力基础上构建的：各个层级的支出上限、访问权限与模型路由、带导出功能和 Analytics API 的使用分析仪表盘，以及努力级别（effort）控制。更丰富的分析能力和更细粒度的成本控制，是我们持续数月打造的这套控制体系中最新加入的部分。

追踪采用情况与成本

面向管理员的分析仪表盘，现在可以按组和按用户展示使用情况与成本，诸如创建的 artifacts、编辑的文件、使用的技能和连接器等产出指标，会直接显示在对应成本旁边。管理员可以按 IT 团队已经在管理的 SCIM 组进行筛选，让这份细分数据直接对应现有的组织架构。

Claude Code 在管理控制台内新增了两个聚焦于价值和使用情况的标签页，带来更丰富的洞察。"使用情况"标签展示全组织范围内的活跃开发者数量、会话次数和高频命令，并每日更新。"价值"标签则汇总使用与成本数据，帮助管理员一眼看懂 Claude Code 带来的价值，估算生产力提升、每次提交的成本，以及年度价值。标签页中的每一条公式都是可见的，输入参数也可以调整。

Analytics chat（分析对话）现在可以回答范围更广的问题，并生成可以进一步深入研究的更丰富的 artifacts。管理员可以用日常语言提问——比如"哪些团队本月的 Claude 使用量翻倍了？"或者"我们在哪些坐席上获得的价值最高？"——Claude 会返回可导出、可分享给利益相关方的图表。

使用量和成本数据现在可以通过 Analytics API 以编程方式获取，因此财务和 IT 团队可以把 Claude 的使用与成本数据引入他们已经在使用的工具——比如 Datadog Cloud Cost Management 和 CloudZero——和其他云与 AI 支出放在一起查看。结果可以按日期范围、团队、产品或模型筛选。各项技能会各自上报自己的使用量和成本，新的端点还能追踪插件的采用情况和 artifact 的创建情况。

管理员可以把使用可见性延伸到单个用户——成本、产品和模型细分，以及相对于支出限额的进度——这样没有人会遇到意外的额度中断。用户也可以查看自己随时间变化的使用趋势，包括自己最依赖哪些产品、模型和技能，以及这些活动累积起来的支出情况。

用于管理支出的控制工具

"模型默认值与权限管理"让管理员可以设置聊天、Cowork 和 Claude Code 中新对话默认使用哪个 Claude 模型，这样常规工作就不会默认套用最贵的选项。管理员可以控制哪些模型对特定角色开放，或对整个组织开放。

支出阈值提醒会在组织级支出限额达到 75% 和 90% 时通知管理员，让他们有时间在任何人被中途拦截之前提高上限。用户会在达到 75% 和 95% 阈值时收到应用内通知，并且无需离开 Claude 就能直接向自己的管理员申请提高限额。

对于需要在众多组之间管理限额的组织，Admin API 可以把成本控制工作流转化为脚本，让控制能力随组织规模一起扩展。可以大规模自动化处理提额申请审核、识别接近支出限额的成员，以及标记使用量快速变化的情况。

"成本可见性不该是一件每月才做一次的事。精细化的支出数据和提醒，能持续提醒团队重新审视自己是如何使用 Claude 的，而不是等到账单周期结束时才大吃一惊。有了 Analytics API，我们可以把这些数据直接引入我们每天都在用的工具里。"
—— Kyra Abbu，产品经理（Workato）
"我不会让那些正在创造公司最佳季度业绩的人慢下来，我的 CFO 也没有要求我这么做。他要的是投资回报率。我们已经把连接了企业 MCP 服务器的 Claude 和 4% 的营收提升联系了起来，而按团队把成本和业务影响放在一起看，正是我用来让这个论证站得住脚的方法。"
—— Carter Busse，CIO（Nubank）
"单看 token 使用量说明不了太多问题。我真正想看到的，是哪些技能在整个组织里被一次又一次地反复运行——那才是价值的真实信号。"
—— Ciro Yamada，产品总监

开始使用

对于负责在整个组织中管理 Claude 的管理员：可以在管理控制台中查看使用情况和成本细分，按组设置模型默认值和支出限额，并配置支出阈值提醒以提前应对超支。使用量数据可以在管理仪表盘中查看，而 Analytics API 让财务和 IT 团队可以把同样的指标提取到现有的报表系统中。
