Claude Code 现已支持 Artifacts

原文标题：Claude Code now supports artifacts ｜ 发布日期：2026 年 6 月 18 日 ｜ 分类：Product announcements ｜ 产品：Claude Code ｜ 阅读时长：约 5 分钟 ｜ 查看原文

把你正在进行中的工作，预览为一个实时的、可交互的网页——基于你完整会话上下文构建，并可与团队共享。

从今天起，Claude Code 可以把工作进度捕捉为一个 artifact，把 Claude Code 的工作成果转化为实时、可共享的可视化页面——包括 PR 走查、系统讲解、仪表盘和发布清单——并随着你的会话推进而自动更新。

一次 Claude Code 会话，涵盖的场景可能从排查一次故障，到重构一项服务，再到分析数月的数据。Artifacts 把这些工作转化为任何人都可以打开和探索的网页，比如一次 PR 走查、一个可以筛选和排序的仪表盘，甚至是一份随着工作推进自动填写的发布清单。Artifacts 让协作共享的工作变得更容易，团队可以把更多时间花在构建上，少花时间沟通状态更新。

基于你会话的上下文构建

Claude Code 利用你会话的完整上下文——包括你的代码库、你的连接器，以及对话本身——来构建 artifact。一个故障页面可以把失败的测试和代码中背后的函数、来自已连接监控工具的错误激增数据，以及你刚运行的会话中得出的根因分析，全部汇总在一起。使用 artifacts，你无需搭建数据源或搭建基础设施。你只需要提出需求，Claude Code 就会基于已有的内容构建出这个页面。

实时更新的原地刷新页面

当 Claude Code 更新一个 artifact 时，已打开的页面会原地刷新，团队成员在页面发布的那一刻就能看到更新。每一次发布都是同一个链接下的一个新版本，并附带版本历史，你可以随时恢复到之前的版本，一个图库还能让你浏览和管理你创建过的所有 artifacts。

根据我们的内部测试，最常见的使用场景之一是调试。这类场景通常是这样的：一位工程师在站会前发起一次故障调查。Claude Code 通读日志，发布出一个 artifact：一条时间线、可疑的提交记录，以及一张错误率图表。她从页面头部把链接分享给团队。等到站会开始时，Claude 已经随着调查的推进两次重新发布了这个页面，纳入了最新的信息。有了 artifacts，团队成员和利益相关方不再需要"给我们讲讲智能体发现了什么"，因为大家看到的是同一个视图，拥有同一份上下文。

仅对你的组织私密可见

每一个 artifact 默认只对其创建者私密可见。当你准备好后，可以直接从页面上把它分享给你的团队成员和你的组织。Artifacts 只能被你所在组织内经过身份验证的成员查看，无法被设为公开。管理员通过一个组织级别的开关和基于角色的范围限定来管理访问权限，设置留存策略，并通过合规 API 获得全组织范围的可见性。

开始使用

直接向你的会话请求一个 artifact——或者干脆要求做点可视化的东西，以下是按角色划分的一些灵感：

  • 法务／开源： 一份针对每一个第三方依赖的许可证审计，从代码仓库直接生成，并标记出任何 copyleft 许可证。"构建一个 artifact，列出每一个第三方依赖及其许可证，标记出任何 copyleft 的部分。"
  • 隐私： 一份数据流向图，展示代码中个人数据被采集、存储和记录的位置。"追踪我们在代码库中触及个人数据的地方，做成一个 artifact 用于隐私评审。"
  • 安全： 直接链接到具体代码行的发现清单，让修复方案清晰明确。"把这次评审中的身份验证相关发现做成一个 artifact，每一项都链接到对应代码。"
  • FinOps／平台财务： 从你的基础设施即代码（infrastructure-as-code）中映射出云资源和成本驱动因素。"从我们的 Terraform 配置中把云资源映射成一个 artifact，按服务分组，并标出主要的成本驱动项。"
  • 软件工程师： 一份评审者真正能够跟上的 PR 或 bug 走查，取材于 diff 和周边代码。"做一个 artifact，走查这个 PR——包括 diff、思路，以及我测试过的内容。"
  • 设计师与前端工程师： 针对某个界面的多个 UX 方向，全部基于你真实的组件构建，这样你选中的那个方案就是可以直接上线的。"给我一个 artifact，展示这个注册表单的 5 种 UX 变体，基于我们的组件库构建。"
  • 资深工程师与架构师： 一份关于某项服务实际是如何组合在一起的映射图，取自真实的引入关系图，而不是白板手绘。"把支付服务的组合方式从代码中映射成一个 artifact。"
  • SRE 与值班工程师： 一个随着你调查推进而不断生长、最终变成事后总结报告的故障页面。"把这次故障做成一个 artifact——时间线、可疑提交、来自我们监控系统的错误激增数据——并随着我调查的推进重新发布。"
  • 工程经理： 一份基于已合并 PR 生成的、真正上线内容的页面。"根据本周我团队合并的 PR，做一个 artifact，按项目分组。"

Claude Code 会构建这个页面，并给你一个链接。可以在浏览器或桌面应用中打开它，从页面头部进行分享——更新会自动发布到同一个 URL。

可用性

Artifacts 目前已面向 Claude Team 和 Enterprise 组织，在 Claude Code CLI 和桌面应用中开放测试版，生成的页面可以在任意浏览器中查看。
