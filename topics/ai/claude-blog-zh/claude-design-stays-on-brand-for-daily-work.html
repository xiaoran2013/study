Claude Design 现已能在日常工作中始终保持品牌一致性

原文标题：Claude Design now stays on brand for daily work ｜ 发布日期：2026 年 6 月 17 日 ｜ 分类：Product announcements ｜ 产品：Claude Design、Claude Code ｜ 阅读时长：约 6 分钟 ｜ 查看原文

Claude Design 现在能够在各个项目之间始终遵循你的设计系统，与 Claude Code 顺畅协同工作，支持直接在画布上编辑，并能连接更多你已经在使用的工具。它在 Claude 桌面应用的侧边栏中有了新的入口；你也可以通过 claude.ai/design 访问它。

我们的目标和之前一样：让你能够探索比截止日期通常允许的更多方向，并让从创意到打磨或构建的交接过程尽可能顺畅。这个目标现在服务着更多的人。上线第一周，就有超过一百万人使用了 Claude Design，你们的反馈正在塑造我们接下来要构建的内容。

用你的设计系统组件进行构建

我们重新构建了设计系统导入功能，带来更强的灵活性和更高的精确度。你可以从一个 GitHub 仓库、设计文件或原始上传文件中，导入一个或多个设计系统。Claude 会用你的组件进行构建，对照你的设计系统检查输出结果，并在你看到之前先行修正。对于更大规模的团队，一个新的管理员角色可以批准一套标准系统并锁定编辑权限，确保工作始终符合公司规范。

在设计与代码之间自由切换

现在，在 Claude Design 和 Claude Code 之间切换工作变得更加容易，同时保持工作内容同步，相关更新从今天起陆续上线。使用 /design-sync 拉取你的设计系统，这样你在 Claude Design 中构建的一切，都会从你现有的组件出发。当一个设计准备好变成软件时，你可以把它交接给 Claude Code，后者会从你已有的工作成果继续推进，而不是从一张截图从头开始。

更喜欢从 Claude Code 开始？你可以使用 /design，无需离开终端，就能创建、编辑和同步设计项目。把一个设计导入你的代码库，把你的代码变成一个可运行的实时原型，或者让 Claude 把一个项目一路承接到底。

更稳定的日常工作体验

我们全新的编辑器让你能够对设计中的每一个元素进行直接、精细的控制。全新、丰富的布局控制功能，让你可以拖动、调整大小和对齐元素。数百项稳定性修复，让编辑器能够经受住真实使用场景的考验。

Claude Design 现在还与聊天、Claude Cowork 和 Claude Code 共享使用额度，因此大多数人将获得更充裕的创作空间，而那些确实会触及限额的人，触及的频率也会降低。现在，平均每一轮对话所消耗的 token 数量更少，却能达到同样的效果，错误率也大幅下降。

从画布直达你常用的工具

可以稳定地导出为 PDF 和 PowerPoint 格式，或者把你的作品发送到你已经在使用的应用中——目前连接器列表已包括 Adobe、Base44、Canva、Gamma、Lovable、Miro、Replit、Vercel 和 Wix，未来还会加入更多目的地。

开始使用

Claude Design 目前在 Claude Pro、Max、Team 和 Enterprise 套餐中开放测试版，包含在你的订阅之中。对于 Enterprise 用户，它默认处于关闭状态；管理员可以在组织设置中启用它，相关工作仅能在你的组织内部共享。可以在 claude.ai/design 或桌面应用的侧边栏中找到它。导入你的设计系统，开始创作。没有灵感？可以让它为你下一个落地页提供三种设计方向，然后挑选一个值得进一步打磨的方案。
