## 文章在讲什么

这篇文章是 Claude Code 团队写的实践指南，主题是如何用**循环（Loops）**让 AI 代理自动重复工作，直到达成目标为止。

作者观察到，现在很多人都在讨论"设计循环"而不是"写提示词"，但网上对这个概念的解释五花八门。所以这篇文章的目标很明确：**给出一种清晰的定义和分类方法**，帮助开发者理解循环是什么、有哪些类型、什么时候该用哪种。

整篇文章的逻辑是：从最简单的场景出发，介绍四种循环类型，然后告诉你怎么选择、怎么开始尝试。它强调的核心原则是：**从最简单的方案开始，不要过度设计**。

## 核心概念讲解

### 什么是循环（Loop）？

文章给循环下了一个定义：

> 循环 = **代理重复工作周期，直到满足停止条件为止**。

这个定义里有三个关键词需要拆开理解：

- **代理（Agent）**：在 Claude Code 里指的是 AI 模型在执行任务时的角色，它可以接收指令、执行动作、返回结果。
- **重复工作周期**：代理不是执行一次就结束，而是会多轮执行，每轮可能做不同的事情。
- **停止条件**：循环不会无限进行，必须有一个明确的退出点。

类比一下：就像你让一个实习生帮你处理文件，你会告诉他"做完这 100 份就停下"或者"处理到没有新文件为止"。循环就是把这个授权过程自动化。

### 循环的四个维度

文章说，区分不同循环类型要看这四个方面：

| 维度 | 含义 |
|------|------|
| 触发方式（How triggered） | 循环在什么情况下开始 |
| 停止方式（How stopped） | 循环在什么情况下结束 |
| 使用的原语（Claude Code primitive） | 具体用哪个命令或工具来实现 |
| 适合的任务类型 | 哪种工作最适合这种循环 |

理解这四个维度很重要，因为它们决定了**你在循环中需要"交出什么控制权"**。文章后面就是围绕这四个维度展开的。

### 核心心智模型："你交出什么？"

作者在四种循环的对比中，用了一个贯穿始终的问题：**"你可以把哪一部分工作交给代理？"**

- Turn-based：你把**检查工作**交出去
- Goal-based：你把**目标定义**交出去
- Time-based：你把**触发时机**交出去
- Proactive：你把**整个提示**交出去

这个思考框架帮助你在实际工作中快速判断该用哪种循环。

## 文章主体讲解

### 引言部分：为什么循环值得学

**这一节讲什么**：作者先抛出一个观察——现在大家都在谈"设计循环"而不是"写提示词"，但网上对循环的解释各不相同。然后他给出了 Claude Code 团队的定义，并预告了文章会覆盖的内容。

**为什么重要**：这解决了读者可能存在的困惑——"循环到底是什么？我在网上看到好多说法。"有了统一定义，后面才能展开讨论。

**关键点**：

- 循环的定义是清晰的，不是模糊的概念
- 文章会介绍主要类型、如何选择、如何在保持代码质量的同时控制 token 消耗
- 但不要滥用循环：**从最简单的方案开始**

作者最后这句话很关键。它提醒我们，循环是高级工具，不是所有任务都需要它。这为整篇文章定下了"实用主义"的基调。

### 四种循环类型

文章用一张表格介绍了四种循环。为了更好地理解，我逐个展开：

#### 1. Turn-based loops（基于轮次的循环）

- **触发方式**：你主动发起下一轮
- **停止方式**：你判断"可以了"才停下
- **使用场景**：探索性任务、不确定要做什么、需要做决策
- **对应工具**：Custom verification skills（自定义验证技能）

这种循环最像我们日常使用 AI 的方式：你问一句，AI 答一句，你觉得满意了就说"可以了"。区别在于，这里 AI 可以多轮执行，而且每次执行后会等待你检查。

适合的场景是当你还在**探索问题**或者**需要判断**的时候。比如你不确定代码改对了没有，需要人工介入确认。

#### 2. Goal-based loops（基于目标的循环）

- **触发方式**：代理自动继续
- **停止方式**：达成目标后自动停止
- **使用场景**：目标明确、知道"完成"长什么样
- **对应工具**：`/goal` 命令

这是最符合直觉的循环类型：你告诉 AI "把这个功能写完"，AI 会一直工作直到他认为完成了。你不需要逐轮检查，只需要定义好目标即可。

关键是你必须有**清晰的目标**。如果目标模糊，AI 可能无法正确判断何时停止。

#### 3. Time-based loops（基于时间的循环）

- **触发方式**：定时触发（比如每天早上、每小时）
- **停止方式**：按计划停止
- **使用场景**：外部工作、定时任务
- **对应工具**：`/loop`、`/schedule`

这种循环的特点是**工作发生在你的项目外部**，并且按时间表运行。比如定期拉取数据、定期发送报告等。

它的核心价值是让你从"记得去做"这件事中解放出来，交给定时器触发。

#### 4. Proactive loops（主动式循环）

- **触发方式**：AI 根据情况自己决定
- **停止方式**：任务完成或条件不满足
- **使用场景**：重复性且定义明确的任务
- **对应工具**：以上所有，以及 dynamic workflows（动态工作流）

这是最"智能"的循环。你把完整的提示交出去，AI 会自主判断什么时候该做什么。它适用于 recurring（周期性发生）且 well-defined（定义清晰）的工作。

作者把这种放在最后，可能暗示它是最复杂的类型，需要对前三种有充分理解后再使用。

### 如何开始：行动指南

**这一节讲什么**：作者给出了从零开始尝试循环的具体步骤。

**为什么重要**：理论和实践之间有鸿沟，这一段把"知道循环是什么"变成"真的用起来"。

**关键点**：

1. **从你现有的工作入手**：不要凭空想象一个场景，找一个你现在就在做、且你是瓶颈的任务。
2. **问自己一个关键问题**：哪部分工作我可以交出去？是可以写验证检查？还是目标已经够清晰？还是工作会按时到来？
3. **先跑起来**：运行循环，观察结果，注意它在哪里卡住或者超出范围。
4. **迭代**：不要期望一次就完美，根据观察调整。

这个方法论很务实——从小处着手，观察反馈，逐步完善。它的本质是把循环设计当成一个实验，而不是一个计划。

## 关键对比表

原文的核心就是这张对比表，我把它重新整理得更清晰：

| 循环类型 | 你交出的部分 | 触发方式 | 停止方式 | 适合场景 | 典型命令 |
|----------|--------------|----------|----------|----------|----------|
| Turn-based | 检查权 | 你手动触发 | 你判断 | 探索性、决策性任务 | Custom verification skills |
| Goal-based | 停止条件 | 自动继续 | 目标达成 | 目标清晰的任务 | `/goal` |
| Time-based | 触发时机 | 定时触发 | 按计划 | 外部工作、定时任务 | `/loop`、`/schedule` |
| Proactive | 整个提示 | AI 自主决定 | 任务完成 | 重复性、定义明确的任务 | 上述所有 + dynamic workflows |

这张表的核心价值在于帮助你**做选择**：面对一个任务，你先问自己"我可以放弃哪部分控制权"，然后对应到合适的循环类型。

## 关键句短摘译

1. **"Agents repeating cycles of work until a stop condition is met."**
   - 中文：代理重复工作周期，直到满足停止条件。
   - 重要性：这是循环的核心定义，后面所有内容都围绕它展开。

2. **"Pick one task where you're the bottleneck"**
   - 中文：找一个你是瓶颈的任务。
   - 重要性：给出了选择循环目标的具体标准——你不应该是那个拖慢进度的人。

3 **"Don't be afraid to iterate on it."**
   - 中文：不要害怕迭代它。
   - 重要性：鼓励读者试错，不要追求一步到位。

4. **"Start with the simplest solution and use these patterns selectively."**
   - 中文：从最简单的方案开始，有选择地使用这些模式。
   - 重要性：这是全文的行动原则，防止过度设计。

5. **"Observe the results like where it stalls or over-reaches"**
   - 中文：观察结果，比如它在哪里卡住或超出范围。
   - 重要性：给出了循环运行后应该关注什么。

6. **"You're exploring or deciding"**
   - 中文：你在探索或做决策。
   - 重要性：明确了 Turn-based 循环的最佳使用场景。

7. **"The work happens outside your project on a schedule"**
   - 中文：工作在项目外部按计划进行。
   - 重要性：解释了 Time-based 循环的独特价值。

8. **"The work is recurring and well-defined"**
   - 中文：工作是重复性的且定义明确的。
   - 重要性：这是 Proactive 循环的前提条件。
