# 项目总纲

## 1. 核心定位

这是一个给个人使用的 **AI 驱动自动化内容生成与学习沉淀系统**。

它不是普通博客，也不是单纯的静态网站。它更像一个小型个人内容工厂:

- AI 负责搜索、阅读、抽取、翻译、总结和生成草稿。
- 本地脚本负责调度、校验和生成页面数据。
- 静态网站负责把结果组织成可浏览、可复习、可追溯的学习工作台。
- GitHub 负责公开静态站点、版本管理、远程备份和 GitHub Pages 发布。
- 飞书负责私有知识库、任务队列、协作记录和过程型内容管理。

当前主题是 AI、英语、学科、投资。内容会陆续增加，后续新增主题会复用同一套模式: 自动收集资料, 结构化处理, 生成学习材料, 发布到个人网站。

## 2. 项目要解决的问题

个人长期学习的难点不只是"找资料", 而是把资料变成能持续复用的系统。

| 问题 | 具体表现 | 本项目的处理方式 |
| --- | --- | --- |
| 信息分散 | 新闻、文章、论文、网页材料分布在不同地方 | 通过 AI 搜索和抓取统一收集 |
| 整理成本高 | 手动复制、摘录、排版很难长期坚持 | 用脚本和 AI 自动生成结构化材料 |
| 学习难复盘 | 看过、问过、整理过, 但之后很难找回 | 统一落盘为 Markdown、JSON、HTML |
| 质量难控制 | AI 生成内容可能不可追溯或混入幻觉 | 保留来源和处理过程 |
| 发布麻烦 | 本地文件不方便随时查看 | 推送 GitHub 后自动发布到 Pages |
| 过程管理混乱 | 草稿、待办、复盘和私有材料不适合全部放公开仓库 | 用飞书承载私有知识库、任务和审核流 |

## 3. 使用者和使用场景

这个系统首先服务个人学习, 不以公开内容平台为目标。

主要使用场景:

- 把 AI 文章、模型笔记、Prompt 和 Agent 工作流沉淀成专题材料。
- 把英语表达、阅读摘录、写作句式整理成复习素材。
- 把小学、初中各学科知识点整理成预习、复习和错题材料。
- 把读过的投资类书籍（如 ETF、基金等产品和策略）整理成结构化策略笔记。
- 在 GitHub Pages 上随时浏览公开材料和学习路线。
- 在飞书里管理私有草稿、待办、复盘、资料收集和内容审核。

## 4. 双平台架构

本项目采用 **GitHub Pages + 飞书** 的双平台架构。

一句话分工:

```text
GitHub = 公开、静态、可版本化、可在线浏览
飞书 = 私有、动态、可协作、可任务化管理
```

### GitHub 放什么

GitHub 仓库和 GitHub Pages 承载"可以公开、适合长期浏览、需要版本记录"的内容。

默认放入 GitHub:

- 静态网站页面: `index.html`, `ai.html`, `english.html`, `k12.html`, `investment.html`, `doc.html`。
- 前端资源: `_assets/*.css`。
- 可公开项目文档: `README.md`, `PROJECT_BACKGROUND.md`, `DIRECTORY_STRUCTURE.md`, `AGENTS.md`。
- 可公开学习内容: `topics/`。
- 无密钥的自动化源码和发布配置: `scripts/`, `.github/workflows/pages.yml`, `.nojekyll`。

GitHub 不应该放:

- 真实 API key、token、私钥、`.env`。
- 原始抓取内容、模型对话、调试日志和未脱敏中间产物。
- 不适合公开的个人计划、私有复盘、账号信息和完整过程记录。

### 飞书放什么

飞书承载"私有、动态、需要管理流程、可能不断编辑"的内容。

默认放入飞书:

- 私有知识库: 原始阅读材料、摘录、长文草稿、学习复盘。
- 任务队列: 今日学习、待读文章、待复盘主题。
- 内容审核: AI 生成草稿、人工确认、发布前检查和修改意见。
- 多维表格: 学习主题库、文章队列、复习计划、标签体系。
- 协作记录: 与 agent 的操作记录、周期总结、问题清单。
- 私有附件: PDF、截图、表格、会议纪要、暂不公开的素材。

飞书不作为公开网站的唯一入口。需要稳定在线浏览的内容, 最终仍应沉淀成静态文件并发布到 GitHub Pages。

### 同步边界

推荐同步方向:

1. **飞书到 GitHub**
   - 飞书中经过确认的学习笔记、专题框架, 可以导出或生成 Markdown/HTML, 进入 GitHub Pages。

2. **GitHub 到 飞书**
   - GitHub Pages 的发布结果、更新摘要, 可以写回飞书作为复盘和任务记录。

3. **本地到两边**
   - 本地脚本和 AI agent 负责生成内容、校验质量, 再分别写入 GitHub 或飞书。

同步原则:

- 公开展示走 GitHub。
- 私有过程走飞书。
- 先审核后发布。
- GitHub 只接收脱敏后的稳定内容。
- 飞书可以保存更完整的过程信息, 但也不能保存真实密钥。

## 5. 系统分层

| 层级 | 主要内容 | 作用 |
| --- | --- | --- |
| 素材层 | `_sources/`（本地, 不进 Git） | 保存 AI 处理用的原始材料 |
| 处理流水线 | `scripts/` | 收集、翻译、生成 |
| 内容沉淀层 | `topics/` | 保存专题笔记和学习路线 |
| 展示层 | `_assets/*.css` | 给静态页面提供样式 |
| 网站页面层 | `index.html`, `ai.html`, `english.html`, `k12.html`, `doc.html` | 提供浏览、阅读和主题入口 |
| 私有工作台层 | 飞书 Docs, Wiki, Base, Task, Drive | 管理私有知识、任务、草稿、审核和复盘 |
| 发布版本层 | Git, GitHub, GitHub Pages | 保存历史、推送远端、自动部署公开在线站点 |

## 6. 内容生成流水线

一条标准内容链路如下:

1. **发现**
   - 从个人学习需求中发现值得处理的主题或材料。

2. **收集**
   - 使用 AI 工具搜索文章、网页或其他材料。

3. **加工**
   - 翻译、摘要、结构化整理成可复习的材料。

4. **生成**
   - 输出专题笔记、学习路线或复盘材料。

5. **发布**
   - 公开内容更新静态页面, 提交 Git, 推送 GitHub, 由 Pages 在线发布。
   - 私有过程、待办和复盘写入飞书。

6. **复习**
   - 在网站上浏览公开沉淀结果。
   - 在飞书里管理下一步学习、修订和审核。

## 9. 关键约束

### 可追溯

AI 生成内容应尽量保留来源。不能只留下最终文章。

### 可回滚

所有有效修改都要提交到 Git。线上内容由 GitHub Pages 从 `main` 分支部署。

### 公私分离

GitHub 面向公开浏览, 飞书面向私有管理。任何内容进入 GitHub 前, 都应默认视为可能被公开访问。

### 不提交密钥

真实 API key、token、私钥和 `.env` 文件不能提交。文档里可以提变量名, 但不能出现真实值。

### 面向个人学习

页面和内容首先服务个人复习、检索和决策, 不追求公开媒体式表达。

## 10. 发布方式

本项目的公开浏览入口是 GitHub Pages:

```text
https://xiaoran2013.github.io/study/
```

默认发布流程:

```bash
git add <公开站点需要的文件>
git commit -m "描述本次变化"
git push
```

推送到 `main` 后, GitHub Actions 会构建 `.pages` 发布包并触发 Pages 部署。通常等待 1-3 分钟后线上生效。

飞书不是 GitHub Pages 的替代物, 而是私有工作台。
